38  // charm++/manual.html, Sec. "Structured Control Flow: Structured Dagger".

40  // High-level overview of the dependency and asynchronous call structure
41  // ---------------------------------------------------------------------
42  //
43  // Directed Acyclic Graph (DAG):
44  // -----------------------------
45  // DAG legend:
46  // Diag -- Rep Diag - compute diagnostics
47  // / Eval - evlaute time step
48  // Eval -- Rep - Output one-liner report
49  //
50  // Interpretation of the above DAG
51  // -------------------------------
52  // Control flow is from left to right and top to bottom.
53  //
54  // Outputing a one-liner report (Rep) can only be done if both the
55  // diagnostics have been collected (Diag) as well as the time step has
56  // been evaluated (Eval). The report is only output at the final time step
57  // stage. Collecting diagnostics if optional, i.e., it does not happen
58  // every time step. If diagnostics are collected trigger_diag_complete()
59  // is signaled from the reduction target, Conductor::diagnostics(). If
60  // diagnostics are not collected in a time step, collection is skipped,
61  // and trigger_diag_complete() is signaled from the shortcut function,
62  // Conductor::diagcomplete(). Either way, the above DAG and its logic
63  // remains the same.
64

66  { when trigger_diag_complete(), trigger_eval_complete() serial "eval"
67  { if (m_stage == 1) report(); } };
