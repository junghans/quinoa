97  //! Const ptr to physical variable access dispatch
98  //! \details Public interface to the first half of a physical variable
99  //! access. cptr() and var() are two member functions intended to be used
100  //! together in case when component and offset would be expensive to
101  //! compute for data access via the function call operator, i.e., cptr()
102  //! can be used to pre-compute part of the address, which returns a
103  //! pointer and var() can be used to finish the data access using the
104  //! pointer returned by cptr(). In other words, cptr() returns part of the
105  //! address known based on component and offset and intended to be used in
106  //! a setup phase. Then var() takes this partial address and finishes the
107  //! address calculation given the unknown id. Thus the following two data
108  //! accesses are equivalent (modulo constness):
109  //! * real& value = operator()( unk, comp, offs ); and
110  //! * const real* p = cptr( comp, offs ); and
111  //! const real& value = var( p, unk ); or real& value = var( p, unk );
112  //! Requirement: offset + component < nprop, enforced with an assert in
113  //! DEBUG mode, see also the constructor.
114  //! \param[in] component Component index, i.e., position of a scalar within
115  //! a system
116  //! \param[in] offset System offset specifying the position of the system of
117  //! equations among other systems
118  //! \return Pointer to data of type tk::real for use with var()
119  //! \see Example client code in Statistics::setupOrdinary() and
120  //! Statistics::accumulateOrd() in Statistics/Statistics.C.
